Contextual retrieval
Pipeline connected. Ask your question in Arabic (MSA), French, Darija, or Arabizi. The pipeline detects your language automatically, translates if needed, retrieves from both indexes, generates via Gemma 4 E4B, and verifies with 11 grounding layers before returning an answer.
0
⚡ Fast mode — answer streams immediately, verification runs in background
Pipeline — 11 Steps
Offline phase builds the knowledge base once. Online phase runs for every query against the real Ollama + Gemma 4 E4B backend.
Offline — runs once
1
Load & Clean
PyMuPDF extracts text from PDFs. Scanned pages fall back to Tesseract OCR (ara+fra). NFKC normalisation applied.
pymupdf · OCR
2
Chunk Text
Sentences split on Arabic ،؟ and French .!? separators. 400-word chunks, 50-word overlap. Each chunk classified and routed to correct index.
400w · overlap 50
3
Embed & Index
BGE-M3 encodes chunks into 1024-dim vectors. Two FAISS IndexFlatIP + two language-specific BM25Okapi indexes built and saved to disk.
BGE-M3 · FAISS
Online — runs per query
4
Detect Language
5-signal ensemble. Signal 1 checks Darija markers (واش، بزاف، كاين…), Signal 2 Arabizi markers (wach, kifach…), then script ratio, keywords, Lingua + langid.
5-signal
5
Translate Query
Only for Darija / Arabizi. Gemma 4 E4B translates to MSA Arabic so BM25 can match government vocabulary. Original query kept for generation.
Darija → MSA
6
Classify & Route
Gemma classifies into multi-label intents: DOCUMENTS, PROCEDURE, COST, DEADLINE, ELIGIBILITY, LEGAL, COMPARISON, OUT_OF_SCOPE. Sets SIMPLE / MULTIHOP flag.
Ollama · multi-label
7
Hybrid Retrieval
Both indexes always searched. BM25 (0.4) + dense (0.6) fused with RRF. Top 10 ScoredChunks returned. Score below threshold → abstain.
BM25 + dense + RRF
8
Rerank + Compress
bge-reranker-v2-m3 cross-encoder scores top 20 (query, chunk) pairs. Reranked top 5 become the generation context.
bge-reranker-v2-m3
9
Generate — PlannerAgent
FIX 71-75: LLM generates an explicit AgentPlan (steps, tools, rationale) before execution. ToolRegistry dispatches 5 tools: retrieve_kb, lookup_article, calculate_deadline, check_eligibility, search_by_amount. After each step, _reflect() checks completeness and may trigger adaptive retrieval. AgentState.facts replaces flat tuple list.
Ollama · Plan→Execute→Reflect
10
Verify — 11 layers
Claim decomp → NLI per claim (mDeBERTa-XNLI) → entity extract → entity match → relation → chain verify → LLM-judge → confidence → citations → CFI → audit JSON.
mDeBERTa · LLM-judge
11
Output
PipelineResult with answer, language, grounding status, sources, latency, audit trail, and translation flag returned to the GUI.
PipelineResult

1
Claim decomposition
Gemma breaks the answer into atomic verifiable facts via structured prompt.
LLM
2
NLI per claim
mDeBERTa-XNLI entailment check per claim vs top 5 context chunks. Multilingual AR + FR.
≥ 0.75
3
Entity extraction
Regex extracts dates (Western + Arabic numerals), amounts (MAD/DH/درهم), Moroccan cities.
regex
4
Entity type verification
Exact string match in chunk texts with Arabic ↔ Western numeral conversion.
100% exact
5
Relation preservation
Entity co-occurrence in context verifies relationships (amount ↔ service) hold.
implicit
6
Multi-hop chain
Each intermediate answer independently NLI-verified. Chain score = 1.0 if all pass.
MULTIHOP
7
LLM-as-judge
Claims with NLI < 0.65 re-evaluated by Gemma: SUPPORTED / PARTIALLY / NOT_SUPPORTED / CONTRADICTED.
< 0.65
8
Confidence scoring
Average confidence computed per claim, stored in audit metadata.
avg
9
Citation validation
[Source: file.pdf] injected on grounded lines, validated against retrieved sources. Invalid → UNVERIFIED.
source set
10
Composite Fidelity Index
CFI = 0.60 × entity_ratio + 0.40 × chain_score. Must be ≥ 0.70.
≥ 0.70
